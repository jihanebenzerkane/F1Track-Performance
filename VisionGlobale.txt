# F1Track — Project Roadmap & Technical Vision

## Core Objective
To develop a high-performance, data-driven platform for Formula 1 analytics, bridging the gap between historical racing data and predictive performance modeling.

## 1. Technical Foundation
- **Database Architecture:** Transitioned from SQLite/flat-files to a robust MySQL 8.4 schema (F1db).
- **Backend Services:** Spring Boot REST API layer handling complex SQL joins, scoring logic, and telemetry proxies.
- **Frontend Layer:** High-fidelity React application utilizing WebGL (Three.js) for 3D car visualization and Recharts for data telemetry.

## 2. Feature Roadmap

### Phase 1: Data Maturity (Standings & History)
- Integration of 70+ years of Formula 1 history via the standard F1db schema.
- Optimized DAO layer for rapid retrieval of driver/constructor standings.
- Real-world mapping of circuits, race results, and qualifying data.

### Phase 2: Technical Deep Dive (Car Viewer)
- 3D interactive car models with granular component analysis (Power Unit, Aero, Tires).
- Direct backend mapping of technical specifications linked to specific car models (e.g., Mercedes W15, Red Bull RB20).

### Phase 3: Telemetry & Live Data
- Proxying real-time session data from the OpenF1 API.
- Live lap timing visualizations and sector-by-sector performance delta analysis.

### Phase 4: Prediction & Strategy Simulation
- Implementation of Monte Carlo simulation engines for race forecasting.
- Driver form index analysis using historical performance weights.
- Pit strategy optimization based on tire degradation and track temperature models.

## 3. UI/UX Principles
- **Aesthetic:** High-contrast, dark-mode, sports-racing theme.
- **Performance:** Optimized 3D rendering and fast asynchronous data fetching.
- **Usability:** Data-rich layouts with strong typographic hierarchy.

## 4. Maintenance & Scaling
- Continuous integration of new season data (2025/2026 regulations).
- Migration of remaining static frontend datasets to the centralized MySQL database.
- Refactoring for modularity and high-performance throughput.
